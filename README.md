Анализ зависимостей Python-пакетов

Проект предназначен для исследования зависимостей Python-пакетов, построения графов зависимостей, поиска обратных зависимостей и их визуализации. Работа выполняется без использования сторонних менеджеров пакетов, что позволяет глубже понять внутренние связи между библиотеками.

Этап 1
На первом этапе реализован ввод данных через XML-файл, извлекаются имя пакета, его версия и URL репозитория. Это создаёт основу для дальнейшей работы с пакетами, так как вся информация для анализа и построения графа зависит от этих данных.

Этап 2
Во втором этапе происходит получение прямых зависимостей пакета. Пакеты загружаются из репозитория, распаковываются метаданные, и из них извлекаются сведения о том, какие библиотеки необходимы для работы пакета. Это позволяет увидеть прямые зависимости без использования pip или других менеджеров пакетов.

Этап 3
Третий этап посвящён построению графа зависимостей с учётом транзитивности. Реализован алгоритм обхода BFS с рекурсией, корректно обрабатываются циклические зависимости. Для тестирования используется граф с буквенными обозначениями пакетов, что помогает демонстрировать работу алгоритма на упрощённом примере. Граф выводится в виде ASCII-дерева для наглядности.

Этап 4
Четвёртый этап добавляет возможность находить обратные зависимости. Пользователь может ввести пакет, и система покажет, какие пакеты зависят от него. Это работает как для тестового графа с буквами, так и для реального графа пакета. Эта функциональность позволяет понять влияние изменений одного пакета на другие.

Этап 5
На пятом этапе реализована визуализация графа зависимостей. Граф преобразуется в формат Graphviz, что позволяет строить наглядные диаграммы с направленными стрелками. Также сохраняется возможность ASCII-вывода для быстрого просмотра в консоли. В результате можно сравнить наглядность диаграммы и текстового дерева зависимостей, а также сопоставить результаты с инструментами стандартного анализа пакетов.
