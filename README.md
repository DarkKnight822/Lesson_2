# Анализ зависимостей Python-пакетов

Проект анализирует зависимости Python-пакетов: строит графы, ищет обратные зависимости и визуализирует их.

## Этапы

1. **Ввод параметров** — через CLI или интерактивный ввод.  
2. **Прямые зависимости** — извлечение `Requires-Dist` из `METADATA` wheel-файлов с PyPI.  
3. **Транзитивный граф** — BFS-обход с ограничением глубины, обработка циклов, ASCII-дерево.  
4. **Обратные зависимости** — поиск пакетов, зависящих от указанного.  
5. **Визуализация** — Graphviz-код: `"A"; "A" -> "B";`.

## Функции и настройки

Скрипт `main_v5.py` работает в двух режимах: реальный и тестовый.  
В реальном режиме загружает только `.whl`, парсит `METADATA`, фильтрует dev-зависимости через условия `extra ==`.  
В тестовом режиме используется встроенный граф (`A→B,C; B→D; C→D,E; E→B`) или `test_repo.txt`.  
Циклы обнаруживаются и помечаются как `(цикл)`.  
Graphviz-вывод всегда содержит **прямые зависимости** — даже при обратном анализе.
